## 4. Осталось / отложено

- **E2E Playwright (план §13)** — ОТЛОЖЕНО по решению владельца (тестирование позже).
  Требуются 4 браузерных сценария: услуга A с этапом II; услуга B через AI-подбор→prefill→
  ветвление→формула; конструктор generate→edit→publish→/take; конфликт ревизий/RBAC/IDOR/
  LLM-unavailable. Ядро этих путей уже покрыто backend/API- и frontend-юнит-тестами;
  недостаёт именно сквозных браузерных. В `docs/traceability.md` помечены как нереализованные.
- **Презентация PDF** — у владельца готова (в репозитории не хранится).
- **Реальные скриншоты в презентации** — 12 скриншотов стенда лежат в `docs/screenshots/`,
  при необходимости берутся оттуда.
- Мелочи вне MVP (план §1): полный KZ-перевод, миграция черновика между версиями
  Definition, файловый антивирус, enterprise BPM, SMS.

## 5. Как запустить / задеплоить

Локально: `cp .env.example .env && make up && make seed` (детали — `README.md`).
Стенд (VM): `git pull --ff-only origin main` → `docker compose up -d --build api web` →
`docker compose run --rm api alembic upgrade head` при новой миграции. Публичный
маршрут — хостовый nginx через `deploy/enable-host-nginx.sh` (запускается вручную, sudo).
Порты наружу — только `127.0.0.1`. `BAITEREK_OPEN_ACCESS=1` держит демо-режим без логинов.
